GitHub Copilot
בהחלט! הנה הסבר ברור על מבנה הדאטאבייס שלך, הקשרים בין הטבלאות, ותיאור של כל תכונה עיקרית:

1. Users (משתמשים)
טבלה ראשית שמייצגת כל משתמש במערכת (הורה, בייביסיטר, אפוטרופוס, אדמין).

id: מזהה ייחודי (UUID).
phone: טלפון.
email: אימייל.
role: סוג המשתמש (PARENT, BABYSITTER, GUARDIAN, ADMIN).
createdAt/updatedAt: תאריכים.
קשרים:

לכל משתמש יכול להיות פרופיל אחד בלבד מסוג ParentProfile, BabysitterProfile או GuardianProfile.
2. ParentProfile (פרופיל הורה)
פרטי ההורה והמשפחה.

id: מזהה ייחודי.
userId: מזהה המשתמש (קשר ל-Users).
full_name: שם מלא.
address: כתובת.
cityId: מזהה עיר (קשר ל-City).
neighborhood: שכונה.
languages: ש.
email: אימייל נוסף.
קשרים:

שייך ל-User (role: PARENT).
שייך ל-City.
יכול להיות לו כמה Requests (בקשות עבודה) ו-Bookings (הזמנות).
3. BabysitterProfile (פרופיל בייביסיטר)
פרטי הבייביסיטר.

id: מזהה ייחודי.
userId: מזהה המשתמש (קשר ל-Users).
full_name: שם מלא.
age: גיל.
cityId: מזהה עיר (קשר ל-City).
neighborhood: שכונה.
walking_radius_minutes: מרחק הליכה בדקות.
service_areas: אזורי שירות.
experience_years: שנות ניסיון.
communityStyleId: מזהה סגנון קהילה (קשר ל-CommunityStyle).
bio: ביוגרפיה.
has_guardian: האם יש אפוטרופוס.
guardianId: מזהה אפוטרופוס (קשר ל-GuardianProfile).
approval_mode: מצב אישור (למשל: אישור לכל בקשה).
languages: ש.
קשרים:

שייך ל-User (role: BABYSITTER).
שייך ל-City.
שייך ל-CommunityStyle.
יכול להיות לו GuardianProfile.
יכול להיות לו Bookings.
4. GuardianProfile (אפוטרופוס)
פרטי אפוטרופוס של בייביסיטר צעיר.

id: מזהה ייחודי.
userId: מזהה המשתמש (קשר ל-Users).
babysitterId: מזהה בייביסיטר (קשר ל-BabysitterProfile).
phone: טלפון.
name: שם.
קשרים:

שייך ל-User (role: GUARDIAN).
שייך ל-BabysitterProfile.
5. City (עיר)
טבלת ערים ושכונות.

id: מזהה ייחודי.
name: שם העיר.
neighborhoods: מערך שמות שכונות.
קשרים:

יכולה להיות משויכת להורים ולבייביסיטרים.
6. CommunityStyle (סגנון קהילה)
סיווג קהילתי (דתי, חילוני, חרדי וכו').

id: מזהה ייחודי.
label: שם הסגנון (למשל: "חילוני").
description: תיאור.
קשרים:

יכולה להיות משויכת לבייביסיטרים.
7. Request (בקשת עבודה)
הורה מפרסם בקשה לבייביסיטר.

id: מזהה ייחודי.
parentId: מזהה הורה (קשר ל-ParentProfile).
datetime_start/datetime_end: תאריכי התחלה/סיום.
area: אזור.
children_ages: גילאי הילדים.
קשרים:

שייך ל-ParentProfile.
יכול להיות לו מועמדים (RequestCandidate).
יכול להוביל ל-Booking.
8. Booking (הזמנה)
הזמנה בפועל של בייביסיטר.

id: מזהה ייחודי.
requestId: מזהה בקשה (קשר ל-Request).
parentId: מזהה הורה.
babysitterId: מזהה בייביסיטר.
datetime_start/datetime_end: תאריכי התחלה/סיום.
address: כתובת.
קשרים:

שייך ל-Request, ParentProfile, BabysitterProfile.
9. TelephonySession (שיחות טלפון)
ניהול שיחות טלפון בין משתמשים.

id: מזהה ייחודי.
call_id: מזהה שיחה.
phone: טלפון.
createdAt: תאריך יצירה.
דיאגרמת קשרים (בקצרה):
User ←→ ParentProfile / BabysitterProfile / GuardianProfile
ParentProfile ←→ City
BabysitterProfile ←→ City, CommunityStyle, GuardianProfile
Request ←→ ParentProfile
Booking ←→ Request, ParentProfile, BabysitterProfile

הסבר:

(PK) = Primary Key (מזהה ייחודי)
(FK) = Foreign Key (קשר לטבלה אחרת)
החצים מייצגים קשרים בין טבלאות (למשל: BabysitterProfile מקושר ל-User, City, CommunityStyle, GuardianProfile)
כל טבלה יכולה להכיל שדות נוספים לפי הצורך.
